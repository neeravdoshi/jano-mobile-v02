## Design Context

### Users
Doctors using a mobile companion app (a PWA, installable to the home screen) across their clinical day. Context: hospital/clinic, time-pressured, glancing between patients — often a single-handed phone glance. Jobs to be done: see the day at a glance, triage urgent patient signals, scan a patient's clinical record (medications, labs, reports, prescriptions), follow pathology trends, manage the merged care-thread chat, and keep the schedule. This is a **prototype** — no backend, all data mocked.

### Brand Personality
Calm · Clinical · Confident. The interface behaves like a trusted senior colleague — it reduces cognitive load, never adds to it. Quiet by default, but unafraid to go bold and red when something is *urgent* or when summarising the day. The crimson is the soul of the brand, not a sprinkle on top.

### Aesthetic Direction

- **Theme — light app, with deliberate dark "spotlight" surfaces.** The app body is warm off-white (`--neutral-app-bg`) with charcoal text, suited to bright clinical settings and quick glances. Against that calm field, two surface families go dark and saturated to signal importance: the **day-summary banner** (`SummaryCard`) and **urgent alerts** (`AlertCard`). These are the focal points, not the whole UI.

- **Palette — the crimson scale used as a *range*, not a single accent.** The earlier "one accent, used sparingly" rule has matured: crimson now spans from light tints to deep gradients, each register with a job:
  - **Light tints** (`--crimson-5/10/20/30`) — chips, insets, soft pills, icon containers.
  - **Base** (`--crimson-base #E54B4B`) — brand identity, primary actions, FAB, active nav, urgent values.
  - **Deep → maroon gradients** (`--crimson-deep` → `--crimson-80` mixed into `--charcoal-base`) — reserved for the dark hero/urgent surfaces below.
  - Charcoal text (`--charcoal-base/50/oslo`) on warm neutrals everywhere else. Clinical status hues (`--status-stable/caution/critical`) carry lab/data meaning.

- **Crimson gradient surfaces (the signature move).** `SummaryCard` is a dark maroon gradient (deep crimson into charcoal) whose inner segment tiles **lift off as translucent-white glass panels** — that layering, not a border, is what separates them. `AlertCard` leans further red (a `crimson-deep → maroon` gradient, white text, white status pill) because it carries *urgent patient alerts*. This is **purposeful depth reserved for prime/urgent surfaces** — it is NOT applied app-wide, and it is gradient *surface fill*, never gradient text.

- **The "ask" AI bar.** The home search is a magical `SearchBar variant="ask"`: a full crimson gradient stroke (border-box gradient, padding-box card) whose colour band drifts in place via Framer Motion, with a leading `Sparkles` glyph and a soft crimson glow. It's the one overtly delightful element — earned because it's the AI entry point.

- **Motion.** Framer Motion is now in the stack (it was previously deferred). Use it *sparingly*, animate transform/opacity only, and always honour `useReducedMotion` (static fallback). First and canonical use: the ask-bar gradient drift.

- **Typography — Figtree only.** A display scale (24/32/48px) for screen heroes and hero numbers; a UI scale (12–20px) for list/card/label content. Never hard-code sizes or weights — always a `type-*` utility class. Documented exception: in-chart font sizes inside `TrendChart` (data-viz needs sub-scale text).

- **Iconography — Lucide only**, 1.5 stroke (2 for emphasis), colour inherited from the parent. No inline SVGs, no second icon set.

- **Data-viz.** `TrendChart` (pathology trends) is the house chart style: a `--status-stable` reference band, dashed gridlines, a grey→status/crimson gradient line, value-labelled points (latest emphasised, out-of-range in crimson), month/day axis. Fluid width via viewBox.

- **Density.** Moderate — denser than a consumer wellness app, calmer than an EMR. Data should be legible at a phone glance.

- **Anti-references.** Generic hospital/EMR software (grey, clunky, form-heavy); consumer health/fitness apps (gamified, over-friendly, neon); startup dashboards (big-metric template, decorative sparklines, gradient cards everywhere). And the universal AI tells: gradient *text*, side-stripe accent borders, glass-everywhere, purple-on-dark.

### Design Principles

1. **Space communicates hierarchy.** Generous gaps between sections, tight grouping within them. Vary spacing for emphasis; never uniform padding everywhere. (All spacing/radius/shadow are tokens — `--space-*`, `--radius-*`, `--shadow-*`; no arbitrary px.)

2. **Crimson is a range with a grammar.** Light tints for quiet surfaces, base for action and brand, deep gradients for the day summary and urgent alerts. Saturation tracks importance — the redder and darker a surface, the more it's saying "this matters." Don't spend deep crimson on the mundane.

3. **Depth is earned, not decorative.** Gradient fills and translucent glass-lift are allowed **only** on the dark hero/urgent surfaces, to convey importance and layering. Everything else stays flat and calm. Never gradient text, never side-stripe borders, never glassmorphism as ambient decoration.

4. **Tokens only, no exceptions.** Every colour is a `var(--color-*)`/crimson/charcoal token, every size a `type-*` class, every gap/radius/shadow a scale token. If a needed value isn't in the scale, raise it — don't hard-code around it.

5. **Mobile-first, inside the phone shell.** The canvas is a 390×844 phone (desktop renders it as a centred phone card with a Dynamic Island; mobile is full-screen with safe-area insets). Every layout must read at that width. Headers clear the island on desktop and the OS status bar on mobile via `.header-safe-top`.

6. **Compose the system, never hand-roll chrome.** Pages assemble atoms/molecules/organisms; shared chrome (headers, composers, list rows, timelines) comes from the component layer. A new pattern is a *variant or prop* on an existing component, not inline markup. And no component is "done" until it's registered in `componentRegistry.ts` **and** has a live preview on `/design-system`.
